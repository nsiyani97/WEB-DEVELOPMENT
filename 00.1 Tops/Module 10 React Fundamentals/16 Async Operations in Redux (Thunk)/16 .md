## 1Ô∏è‚É£ Middleware Concept

### ‚úÖ What Is Middleware?

**Middleware** is a layer that sits **between an action and a reducer**:

```text
Action  ‚Üí  Middleware  ‚Üí  Reducer
```

### üîß What Middleware Can Do

- Intercept actions
- Modify actions
- Delay actions
- Perform async operations (API calls)

---

## üîÑ Redux Without Middleware

```text

```

Only **plain JavaScript objects** are allowed as actions:

```js

  type: "ADD_TASK";

```

‚ùå No async logic
‚ùå No functions allowed

## üîÑ Redux With Middleware (Thunk)


‚úÖ Now we can dispatch **functions**, not just objects.

## üß† Why Do We Need Middleware?

Middleware is useful for:

- Async API calls
- Authentication & authorization checks

## 2Ô∏è‚É£ Redux Thunk ‚Äì Core Idea

### ‚úÖ Definition

**Redux Thunk** is a middleware that allows **action creators to return a function instead of an object**.

---

### üîπ Normal Action Creator (Synchronous)

```js

  type: "ADD_TASK",
  payload: task,
});
```

‚úîÔ∏è Returns an object
‚úîÔ∏è Executes immediately

---

### üîπ Thunk Action Creator (Asynchronous)

```js

  return (dispatch) => {
    // async logic here
  };
};
```

‚úîÔ∏è Returns a function
‚úîÔ∏è Can perform API calls
‚úîÔ∏è Can dispatch multiple actions

---

### ‚ú® Key Rule of Redux Thunk

Thunk provides access to:

```js

```

This allows you to:

- Dispatch multiple actions
- Read current store state
- Wait for API responses before updating the store
